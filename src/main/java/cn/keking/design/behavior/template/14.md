模板方法模式：（行为型模式）
定义：模板方法模式是指定义一个操作中的算法骨架，而将一些步骤延迟到子类中。模板方法使子类可以不改变一个算法的结构。
即可重定义该算法的某些特定步骤。
使用场景：
1、编制一个通用的算法，将某些步骤的具体实现留给子类来实现；
2、需要重构代码，将各个子类的公共行为提取到一个共同的父类中，避免代码重复。

优点： 
1、封装不变部分，扩展可变部分。 
2、提取公共代码，便于维护。 
3、行为由父类控制，子类实现。
缺点：
每一个不同的实现都需要一个子类来实现，导致类的个数增加，使得系统更加庞大

demo:求最大值

应用场景：
1、JDK中，如Arrays类中的排序sort(),实现Comparator接口、二分查找（binarySearch()）
2、spring 中对 Hibernate 的支持，将一些已经定好的方法封装起来，
   比如开启事务、获取 Session、关闭 Session 等，不必重复写那些已经规范好的代码，
   直接丢一个实体就可以保存。
总结：
将通用的提取出来，放在父类中，由父类控制，子类自己实现各自的行为.
