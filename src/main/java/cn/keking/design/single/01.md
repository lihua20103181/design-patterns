单例模式：
单例模式保证一个类仅有一个实例，并提供一个访问它的一个全局访问点，
当系统需要某个类只能有一个实例时，就可采用单例模式。其属于创建型模式
特点：
1、只有一个实例
2、自己创建自己的实例
3、给其他对象提供实例
4、核心思想：构造方法私有化
单例模式实现方式：
1、直接实例化 Singleton1.java
2、延迟实例化 Singleton2.java
   有三种实例化方法（完成同步、部分双重机制同步、静态内部类）
使用总结：
1、当java虚拟机加载应用程序字节码时，单例对象不是立即加载的，而是动态生成。
2、使用静态内部类实现单例效率更好。原因：直接定义了变量，private static final Singleton2 SINGLETON_2 = new Singleton2();这个变量
  在Java虚拟机加载应用程序字节码的时，单例对象直接生成，

单例使用场景：
1、系统日志记录
2、系统读取配置文件（如：读取数据库配置信息）
3、应用服务器单例仿真技术，（如;jsp,Servlet在内存中只有一个实例存在）
4、Spring 依赖注入默认使用的是单例（双重判断加锁机制实现）
